# 一、前端工程化与 Webpack

# (1) 引入与使用

前端的实际开发：模块化、组件化、规范化、自动化。

前端工程化: 把开发所需工具、技术、流程、技术等进行规范化、标准化。

  1. 概念：webpack 是前端项目工程化的具体解快方案。

  2. 主要功能：它提供了友好的前端模块化开发支持，以及代码压缩混淆、处理浏览器端 JavaScript 的兼容性、性能优化等强大的功能。

  3. 好处：让程序员把工作的重心放到具体功能的实现上，提高了前端开发效率和项目的可维护性。

  4. 注意：目前 Vue，React 等前端项目，基本上都是基于 webpack 进行工程化开发的。

    其他：webpack (opens new window) parcel (opens new window)

  5. 使用：（-S 是 --save 的简写；-D 是 --save-dev 的简写）

    • npm i webpack@5.42.1 webpack-cli@4.10.0 -D

    • 创建 webpack.config.js 配置文件

      const path = require('path')

module.exports = {
  // 代表 webpack 运行模式，development 开发阶段，production 上线阶段（可进行压缩）
  mode: 'development',
  // 指定路径打包与输出（文件夹和名称）
  entry: path.join(__dirname, './src/index1.js'),
  output: {
    path: path.join(__dirname, './dist'),
    filename: 'bundle.js',
  },
}
/* webpack4.x 5.x 默认打包路径为./src/index.js ，然后输出为./dist/main.  js。*/

    • 在 package.json 的 script 节点下

      "scripts": {
     "dev": "webpack"
     //在终端运行 npm run dev 可打包构建
     //特别注意：node 必须为 16+ 版本
     //会生成 dist 文件
 },

# (2) 扩展插件

  1. 创建服务

    • 安装：npm i webpack-dev-server@3.11.2 -D

    • 修改：package.json -> script 中的 dev 命令

      "scripts":{
  "dev": "webpack serve"
}

    • 输入：npm run dev

    • 浏览器输入打开：http://localhost:8080 ，查看自动打包效果。

    • 配置：在 webpack.config.js 的 module.exports 节点下

      devServer: {
  open: true,   // 自动打开浏览器
  host: "127.0.0.1",  // 主机地址
  port: 80,   // 打包使用的端口
},

  2. 复制页面到内存

    复制页面到内存，便且自动注入打包好 bundle.js（bundle.js 放在了项目根目录"/"）

    • 安装：npm i html-webpack-plugin@5.3.2 -D

    • 使用：

      // 在 webpack.config.js 中配置

// 1.导入 html-webpack-plugin 插件
const HtmlPlugin = require('html-webpack-plugin')
// 2.插件插件的实例对象
const htmlPlugin = new HtmlPlugin({
  template: path.join(__dirname, './src/index.html'),
  filename: 'index.html',
})
module.exports = {
  // 3.插件数组
  plugins: [htmlPlugin],
}

# (3) Loader

webpack 只会打包 index.js

  1. css 样式表 loader

    • 安装： npm i style-loader@3.0.0 css-loader@5.2.6 -D

    • 配置：在 webpack.config.js 的 module.exports 节点下

      module: {
  rules: [
    // test为文件后缀名的匹配；use数组中 调用顺序为从后往前
    { test: /\.css$/, use: ['style-loader', 'css-loader'] },
  ]
}

  2. less loader

    • 安装：npm i less-loader@10.0.1 less@4.1.1 -D

    • 配置：在 module 下 rules 继续添加

      module: {
  rules: [
    // test为文件后缀名的匹配；use数组中 调用顺序为从后往前
    {
      test: /\.less$/,
      use: ['style-loader', 'css-loader', 'less-loader'],
    },
  ]
}

  3. 图片 loader

    base64 图片 可减少发送请求，但会增大文件体积，适合小图标使用。

    • 安装：npm i url-loader@4.1.1 file-loader@6.2.0 -D

    • 配置：

      module: {
  rules: [
    // 写法一：
    // limit 指定是否转 base64 的临界值 X 字节，outputPath=images 指定输出  路径
    // 多个参数可用&分隔
    {
      test: /\.(png|jpg|gif)$/,
      use: 'url-loader?limit=1000&outputPath=images',
    },

    // 写法二：
    {
      test: /\.(png|jpg|gif)$/,
      use: {
        loader: 'url-loader',
        options: {
          limit: 1000,
          outputPath: 'images',
        },
      },
    },
  ]
}

  4. 打包处理 js 高级语法 loader

    • 安装：npm i babel-loader@8.2.2 @babel/core@7.14.6 @babel/plugin-proposal-decorators@7.14.5 -D

    • 配置：

      {
 // 处理 js 高级语法（exclude 排除项）
 test: /\.js$/,
 exclude: /node_modules/,
 use: {
   loader: "babel-loader",
   options: {
     // 声明一个 babel 插件，此插件用来转化 class 中的高级语法
     plugins: ['@bable/plugin-proposal-properties'],
   }
 },
}

    • 创建 babel.config.js

      module.exports = {
  // 声明 babel 可用的插件
  // webpack 在调用 babel-loader 的时候会先调用 plugins
  plugins: [['@babel/plugin-proposal-decorators', { legacy: true }]],
}

# 二、Vue 基础入门

官网 (opens new window) vue.js devTools 插件下载 (opens new window)

# (1) Vue 的介绍

  1. Vue 是构建用户界面（用 vue 页面填充数据）的框架。

    学习路径：vue 指令、组件、路由、vuex、vue 组件库

  2. vue 的两个特性：

    • 数据驱动视图（单向的数据绑定）

      数据的变化会驱动视图自动更新。

      好处：数据变化时页面重新渲染。

    • 双向数据绑定

      js 数据的变化自动渲染到页面；页面表单采集的数据发生变化会更新到 js 数据中。

  3. MVVM（Model 数据源、View 视图、ViewModel 是 vue 的实例）:

# (2) Vue 的指令

模板语法

  1. 内容渲染指令

    • v-text：会覆盖标签内部的内容。

    {{ }}：插值表达式，只能放在内容节点。
    • v-html：可将字符串渲染为 html 结构。

  2. 属性绑定指令

    v-bind：属性绑定值，可简写为 : 。

  3. 事件绑定指令

    v-on：使用v-on:click,可简写为 @ ，@input、@keyup....

    • $event：原生 DOM 的事件对象，在最后一个参数。

    • 事件修饰符：

    修饰符 说明
    .prevent 阻止默认行为
    .stop 阻止事件冒泡
    .capture 以捕获模式触发当前事件处理函数
    .once 绑定的事件只触发一次
    .self 只有 event.target 是当前元素自身触发事件处理函数(事件委托)

    按键修饰符：.esc、.enter、.a、.b ....

  4. 双向绑定指令

    v-model：只能和表单元素使用。

    修饰符 作用 示例
    .number 自动将用户的输入值转为数值类型 <input v-model.number="age"/>
    .trim 自动过滤用户输入的首尾空白字符 <input v-model.trim="msg"/>
    .lazy 在“change”时而非“input”时更新 <input v-model.lazy="msg"/>

  5. 条件渲染指令

    v-if：动态移除、创建元素，如果展示次数少用这个。

    v-show：动态添加 display: none 样式，频繁使用性能更好。

    同级元素配套指令：v-else、v-else-if

  6. 列表渲染指令

    v-for：使用 v-for="item in list"

    获取索引 (item, index) in list

    便且在 v-for 后面加 :key="item.id" (必须为字符串、数字，不能重复；不能用 index 作为 key 的值，没有意义)

# (3) 过滤器

全局过滤器

// 使用
<p>{{ message | capi}}</p>

// 全局过滤器（在外面）
Vue.filter("capi", (val) => {
    const first = val.charAt(0).toUpperCase();
    const other = val.slice(1);
    return first + other;
});

// 私有过滤器（在vue创建实例时）
filters: {
  capi(val) {
    const first = val.charAt(0).toUpperCase();
    const other = val.slice(1);
    return first + other + "----";
  },
},

全局过滤器与私有过滤器发生冲突时，采取最近原则。

可连续调用多个过滤器；便且可传参，从第二个位置接收(第一个为管道符前面那个)。

# (4) 侦听器

watch 侦听器监视数据变化。（在 watch 节点下）

  1. 方法格式监视器

    无法在刚进入页面时自动触发；如果侦听的是对象，对象属性发生变化时无法侦听到。

    // 方法格式监视器
username(newData, oldData) {
  // console.log("值发生了变化", newData, oldData);
  if (newData === "") return;
  $.get("https://www.escook.cn/api/finduser/" + "newData", (res) => {
    console.log(res);
  });
},

  2. 对象格式监听器

    可通过immediate选项，自动触发。

    info: {
 handler(newData, oldData) {
     console.log("值发生了变化", newData, oldData);
 },
 // 决定是否触发一次
 immediate: true,
 //是否深度侦听
 deep: true,
},
// 只监听单个属性
'info.username'

# (5) 计算属性

定义的时候，被定义为方法；使用时当普通的属性使用

computed: {
    rgb() {
        return `rgb(${this.r}, ${this.g}, ${this.b})`;
    },
},

  • 实现了代码的复用

  • 只要计算属性中依赖的数据源（不包含对象、数组）发生变化，则计算属性会重新赋值。

  • 计算属性无法直接传参，但可以通过闭包函数实现传参。

    computed: {
   computedValue() {
       return function(val) {
           return val + 1
       };
   }
}

WARNING

计算属性和侦听器侧重的应用场景不同： 计算属性侧重于监听多个值的变化，最终计算并返回一个新值； 侦听器侧重于监听单个数据的变化，最终执行特定的业务处理，不需要有任何返回值。

官网 (opens new window)

快速开发单页面应用程序 SPA；Vue.js 开发的标准工具。

# 三、组件与生命周期

vue 是一个组件化开发的框架

组件化开发：提高了组件的复用性、灵活性；提升了开发效率、可维护性。

# (1) 基本使用

  1. 三大组成部分：

    • template：组件的模板结构

    • script：组件的 JavaScript 行为

    • style：组件的样式

    TIP

    在 template 只能有一个根元素。

    style 中如果要用 less，则在 style 中加lang="less"。

  2. 使用组件：

    • 导入需要的组件

      // 在 script 标签内（@表示项目根目录）
import Left from '@/components/Left.vue'

    • 注册

      export default {
  components: {
    Left,
  },
}

      通过 components 注册的是私有子组件。

      注册全局组件：

      // 在 main.js 通过 Vue.component() 方法注册全局组件。
import Left from '@/components/Left.vue'
Vue.component('Left', Left)

    • 以标签形式使用

      <template>
  <div>
    <Left></Left>
  </div>
</template>

# (2) props

组件的自定义属性，提高了组件的复用性。

// 在export export default 内
// 简写：props: ['init']
props: {
    init: {
    type: Number,
    default: 0,
    required: true,
  },
},

  • props 是只读的，不可直接修改值，否则会报错；可通过把值赋值给 data 里面的数据。

  • props 默认为 undefined，可通过添加属性 default 设置默认值；type 属性设置类型(可通过数组设置多种类型)；required 设置为必填项。

# (4) 组件之间的数据共享

  1. 父组件->子组件：自定义属性；插槽；

    展开 
    // 自定义属性 // 父组件
<template>
  <div>
    <Left :msg="msg"></Left>
  </div>
</template>
<script>
import Left from '@/components/left'
export default {
  components: {
    Left,
  },
  data() {
    return {
      msg: 'Hello Son',
    }
  },
}
</script>

// 子组件
<template>
  <div>
    <p>接收到的消息：{{ msg }}</p>
  </div>
</template>
<script>
export default {
  props: ['msg'],
}
</script>

  2. 子组件->父组件：自定义事件；ref；作用域插槽；

    展开 
    // 自定义事件 // 父组件
<template>
  <div>
    <Left @tomsg="getmsg"></Left>
    <p>接收到的消息：{{ msg }}</p>
  </div>
</template>
<script>
import Left from '@/components/left'
export default {
  components: {
    Left,
  },
  data() {
    return {
      msg: '',
    }
  },
  methods: {
    getmsg(val) {
      this.msg = val
    },
  },
}
</script>

// 子组件
<template>
  <div>
    <button @click="send">发送</button>
  </div>
</template>
<script>
export default {
  data(){
    return {
      msg: 'Hello Father'
    }
  }
  methods: {
    send(){
      this.$emit('tomsg', this.msg)
    }
  }
}
</script>

  3. 兄弟组件之间：EventBus；如果是三级，可通过插槽然后转成父组件与子组件的关系然后传值；vuex(下文会详细讲解)。

# (5) 其他

  1. 解决样式冲突问题：

    在 style 中添加scoped；父组件修改子组件样式（使用第三方组件库时） /deep/ h5。

  2. ref

    ref 可以在不依赖 jQuery，快速操作 DOM 元素。

    每个 vue 组件的实例上都包含一个 $refs 对象。

    vue中$ref取值，静态的ref读取到的是dom对象，而for循环生成得ref，读取到的dom对象是一个数组。

  3. 更新数据后重新渲染完毕后执行（DOM 更新是异步的）：

    this.$nextTick(() => {})

  4. 规范写法：先指令、后绑定属性、再绑定事件。

# 四、Vue 组件的高级用法

# (1) 动态组件

  1. vue 自定义了<component>组件，动态切换组件的显示与隐藏，通过 is 属性确定要渲染那个组件。

  2. keep-alive：被包裹的组件不会被销鬼，加入缓存，保持状态。

    生命周期函数：第一次创建时会触发 created 和 activated ，被激活时只触发 activated ，被缓存时触发 deactivated。

    include 属性可指定需要被缓存的组件，用逗号分隔。

    exclude 属性可指定不需要被缓存的组件，与前者不能一起使用。

  3. 如果给组件“声明名称”，就以名称为准，否则以注册名称为准。

    注册名称：应用在使用标签。

    声明名称：调试工具时的名称，<keep-alive>上。

# (2) 插槽

组件标签内的内容自动填充到组件插槽中。

每个插槽都要有 name （具名插槽），默认为 default ，默认情况下提供的内容填充到 default 。

v-slot 放在组件或 <template>，简写为#，可将其指定放在某个插槽。（template 是虚拟的标签，只起到包裹作用）

在<slot>标签中放置插槽默认内容（后备内容）；在<slot>标签添加属性成为了作用域插槽，可实现子组件传父组件。

<!-- 父组件 -->
<Left>
  <template #mySlot1="{msg}">

# (3) 自定义指令

v-color="color"

  1. 私有自定义指令

    directives: {
  color: {
    // binding.value 获取传参
    bind(el, binding) {
      // el.style.color = 'red';
      el.style.color = binding.value;
    },
    update(el, binding) {
      el.style.color = binding.value;
    },
  },

 // 当 bind 和 updete 要处理的逻辑相同时可这么写
 color(el, binding) {
   el.style.color = binding.value;
 },
}

    bind() 函数在指令第一次绑定元素时调用一次；

    update() 函数在 DOM 被更新时调用。

  2. 全局自定义指令

    在 main.js 中加入以下代码

    // 若 bind 与 update 不同则参数2传对象。
Vue.directive('color', (el, binding) => {
  el.style.color = binding.value
})

# 五、路由

指 Hash 地址与组件之间的队友关系。

前端路由监听到 Hash 地址 的变化然后把对应的组件渲染出来。

# (1) vue-router 使用

官网 (opens new window)

  1. 安装（3.x 版本的）：npm i vue-router@3.5.2 -S

  2. 创建路由模块：在 src 目录下新建 router/index.js 路由模块

    // 1. 导入 Vue 和 VueRouter
import Vue from 'vue'
import VueRouter from 'vue-router'

// 2. 将 VueRouter 安装为 vue 插件
Vue.use(VueRouter)

// 3. 创建路由的实例对象
const router = new VueRouter({})

// 4. 向外共享路由实例对象
export default router

  3. 挂载路由

    // 在 main.js
import router from '@/router'

new Vue({
  render: (h) => h(App),
  router,
}).$mount('#app')

  4. 声明路由链接

    // 在router模块
import Login from '@/components/login'

// 路由重定向（强制跳转）：{ path:'/' ,redirect: '/home'}
const router = new VueRouter({
  routes: [
    { path: '/', redirect: '/login' },
    { path: '/login', component: Login },
  ],
})

  5. 使用：

    • 在路由展示区加上占位符：<router-view></router-view>

    • 可以使用<router-link to="/home"></router-link>替换 a 链接。

# (2) 路由嵌套

通过 children 属性

const router = new VueRouter({
  routes: [
    {
      path: '/home',
      component: Home,
      children: [
        { path: '', component: MyUsers },
        { path: 'rights', component: MyRights },
      ],
    },
  ],
})

默认子路由：在 children 数组中 path 值为空的字符串为默认子路由（router-link 中 to 要修改）。

# (3) 动态路由

把 Hash 地址中可变的部分定义为参数项（路径参数）。

  1. {path: '/movie/:id',compentent: Movie}，在 Movie 组件获取 id：·this.$route.params.id

  2. {path: '/movie/:id',compentent: Movie,props:true}，在 props 中可获取 id。

  3. this.$route是路由“参数对象”；this.$router 是路由“导航对象”

    查询参数（?后面的参数）：this.$route.query

    this.$route.path是路径部分，this.$route.fullpath 是完整路径。

# (4) 导航类别

  1. 声明式导航：<a>、<router-link>

  2. 编程式导航（location.href）

    • this.$router.push('')：跳转并增加一条历史 1 记录。

    • this.$router.replace('')：跳转并替换当前页面。

    • this.$router.go(数值n)：在浏览历史中前进后退（n为正数、负数，超过上限则原地不动）。或this.$router.back()、this.$router.forward()。

    • location.reload()：刷新当前页面。

    注意：在行内编程时 this 必须省略。

# (5) 导航守卫

全局前置守卫：

  • 基本知识
// 不声明next，默认允许访问路由
// 声明了next，必须调用 next()，否则不许访问任何路由
router.beforEach((to,from,next){  //目标路由，当前路由，设置权限
	next()  //放行
	next('/login')  //强制跳转
	next(false)  //不跳转
})
  • 使用

路由模块：

import pathArr from '@/pathArr.js'

// 前置路由守卫
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  let rg = /^\/home\/userinfo\/\d?/
  if (pathArr.indexOf(to.path) != -1 || rg.test(to.path)) {
    if (token) {
      next()
    } else {
      next('/login')
    }
  } else {
    next()
  }
})

pathArr.js：

export default [
  '/home',
  '/home/users',
  '/home/rights',
  '/home/goods',
  '/home/orders',
  '/home/settings',
  '/home/userinfo',
]

# (2) Vuex

实现组件全局状态（数据）管理的一种机制，可以方便的实现组件之间数据的共享。

  1. 引入

    好处：

    • 能够在vuex中集中管理共享的数据，易于开发与维护。
    • 能够高效地实现组件之间的数据共享，提高开发效率。
    • 存储在vuex中的数据都是响应式的，能够保持数据与页面的同步。

    一般只有组件之间共享的数据才存入vuex，组件私有数据存入data中。

  2. 使用

    安装：npm i vuex -S

    导包：在 main.js

    import Vuex from 'vuex'
Vue.use(Vuex)

    创建 store 对象

    const store = new Vuex.Store({
    // state 存放的是全局共享的数据
    state: { count:0 }
})

    将 store 对象挂载到 vue 实例中

    new Vue({
    el: '#app',
    render: h => h(app),
    router,
    // 所有组件可直接从store中获取全局数据 
    store
})

  3. 核心概念

    (1) State：提供唯一的公共数据源

    const store = new Vuex.Store({
    state: { count:0 }
})

// 组件访问的方式1
this.$store.state.全局数据名称
// 组件访问的方式2
import { mapState } from 'vuex'
computed: {
    ...mapState(['count'])	// 将全局数据，映射为当前组件的计算属性
}
// this.count

    (2) Mutation：用于变更state中的数据（不可直接修改；使用此方式可集中监控所有数据的变化）

    const store = new Vuex.Store({
    multations: {
        add(state){
            state.count++
        }
        // 传参
        addN(state, step){
            state.count += step
        }
    }
})

// 组件使用的方式1
methods: {
    handle(){
        this.$store.commit('add')
        this.$store.commit('addN', 2)  // 传参
    }
}
// 组件使用的方式2
import { mapMutations } from 'vuex'
methods: {
    ...mapMutations(['add', 'addN'])
}
// this.add()  this.addN(2)

    注：不能在 mutations 中执行异步操作。

    (3) Action：用于处理异步任务；在 Action 中通过触发Mutation的方式间接变更数据

    const store = new Vuex.Store({
    actions: {
    	addAsync(context) {
    		setTimeout({
                context.commit('add')
            }, 1000)
		}
        addNAsync(context, step) {  // 传参
    		setTimeout({
                context.commit('addN', step)
            }, 1000)
		}
	}
})

// 组件使用的方式1
methods: {
    handle(){
        this.$store.dispatch('addAsync')
        this.$store.dispatch('addAsync', 3)  // 传参
    }
}
// 组件使用的方式2
import { mapActions } from 'vuex'
methods: {
    ...mapMutations(['addAsync', 'addNAsync'])
}
// this.addAsync()  this.addNAsync(3)

    (4) Getter：用于对Store中的数据进行加工处理形成新的数据。（类似计算属性，会随着数据源的变化而变化）

    const store = new Vuex.Store({
    getters: {
        showNum: state => {
            return '当前的最新数量为：' + state.count + '。'
        }
    }
})

// 组件使用的方式1
this.$store.getters.showNum
// 组件使用的方式2
import { mapState } from 'vuex'
computed: {
    ...mapState(['showNum'])
}
// this.showNum

  4. 补充：

    • action中可通过 this.state.名字 直接访问state中的数据
    • action函数中调用action函数：this.dispatch('函数名')
