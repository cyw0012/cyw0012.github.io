# TypeScript

# 一、入门引入

  1. TypeScript 是微软开发的，是 JavaScript 的超集，它包含了 JavaScript 的功能，还添加了类型支持，可在任何运行 Js 的地方运行。

  2. 优点：

    • 避免了类型错误，减少 Debug 的时间，提高开发效率。
    • 有代码提示，增加了开发体验
    • 强大的类型系统提升了可维护性，易于重构
    • 支持最新的 ECMAScript 语法
    • Ts 类型推断机制，无需在每个地方标注类型，降低了使用成本

  3. 实现方式：Ts 为静态类型编程语言，Js 为动态类型编程语言。

    静态类型：编译期做类型检查

    动态类型：执行期间做类型检查

    代码执行顺序为先编译->再执行，这样 Js 发现错误的时间就晚了，而 Ts 则可配合开发工具在编写代码的同时发现错误。

  4. 初体验：

    • 安装编译 Ts 的工具包：npm i -g typescript

    • 检查安装(查看版本)：tsc -v

    • 创建 ts 文件：hello.ts

      console.log('hello ts')
let age: number = 18
// age = 'yyds' 会报错
console.log(age)

    • 编译：tsc hello.ts

    • 执行：node hello.js

    简化编译和执行两个步骤(内部将 ts->js 执行，不会生成 js 文件)

    使用 ts-node 包：npm i -g ts-node

    使用方式：ts-node hello.ts

# 二、Ts 的常用类型

Ts 会检查变量类型的变换，从而避免类型改变发生错误。

类型注解：let age: number = 18中的: number即为类型注解，为变量添加了类型约束。

# (1) Js 现有类型

  1. 原始类型：number、string、boolean、null、undefined、symbol

    let num: number = 23
let isLoading: boolean = false
let nameStr: string = 'lihua'
let a: null = null
let b: undefined = undefined
let c: symbol = Symbol()

  2. 对象类型：object（数组、对象、函数）

    • 数组
    let arr1: Array<number> = [1, 2, 3] //不推荐
let arr2: number[] = [1, 2, 3]
let arr3: string[] = ['aa', 'bb', 'cc']
    • 对象
    // 在一行需要加分号，分行可不加
let obj1: { name: string; age: number; sayHi(str: string): void } = {
  name: '小明',
  age: 19,
  sayHi(str) {
    console.log(str)
  },
}
let obj2: {
  name: string
  age: number
  sayHi: (str: string) => void
} = {
  name: '小明',
  age: 19,
  sayHi(str) {
    console.log(str)
  },
}

// 对象可选属性
function myAxios(config: { url: string; method?: string }) {}
myAxios({
  url: '',
})
    • 函数（给参数、返回值设置类型）
    // 3.1 （1）单独指定
function add(num1: number, num2: number): number {
  return num1 + num2
}
const add1 = (num1: number, num2: number): number => num1 + num2
console.log(add1(23, 34))

// 3.1 （2）同时指定（只适于函数表达式）
const add2: (num1: number, num2: number) => number = (num1, num2) =>
  num1 + num2
console.log(add2(12, 13))

// 3.2 返回值为空（可省略）
const handle = (str: string): void => {
  console.log(str)
}
handle('hello')

// 3.3 可选参数（参数可传可不传）
// 在参数名称后面加 ? ，必须在参数列表的最后
// end = 4 设置参数默认值（TypeScript会将添加了默认值的参数识别为可选参数）
function mySlice(start?: number, end = 4): void {
  console.log('起始索引：', start, '结束索引：', end)
}
mySlice()
mySlice(1)
mySlice(1, 5)

# (2) Ts 新增类型

  1. 联合类型（|）

    let arr11: (number | string)[] = [12, 32, 43, 'wqwqwq']
let arr12: number | string[] = ['23', 'wqwqwq']

  2. 自定义类型（类型别名 type）

    type CustomArray = (number | string)[]
let arr21: CustomArray = [12, 32, 43, 'wqwqwq']

  3. 接口（interface）

    接口与类型别名的区别：都可给对象指定类型；但接口不能指定其他类型

    interface IPerson {
  name: string
  age: number
  sayHi(): void
}
// type IPerson = {
//   name: string
//   age: number
//   sayHi(): void
// }
let person1: IPerson = {
  name: 'John',
  age: 34,
  sayHi() {},
}

// 可继承
interface Person2D {
  x: number
  y: number
}
interface Person3D extends Person2D {
  z: number
}
let point3d: Person3D = {
  x: 12,
  y: 14,
  z: 23,
}

  4. 元组（Tuple）

    另一种数组类型，确切的包含多少个元素，以及特定索引对应的类型。

    let tuple: [string, number] = ['yyds', 23]

  5. 字面量类型

    主要用于变量只能是其中的某一个。

    function handle(direction: 'up' | 'down' | 'left' | 'right') {}

  6. 枚举（enum）

    定义一组命名常量

    // 枚举成员有值，默认从0自增长
enum Direction {
  left,
  right,
  up,
  down,
}
function func(direction: Direction) {}
func(Direction.left) // 访问枚举成员

// 从12自增长、给定值（字符串没有自增长，需每个设置）
enum Direction {
  left = 12,
  right,
  up,
  down,
}
enum Direction {
  left = 2,
  right = 4,
  up = 6,
  down = 9,
}

    注意

    字面量+联合类型：更直观、简洁、高效（推荐）

  7. any

    不推荐使用，可对该值进行任意操作，主要用于临时使用。

    let obj: any = { x: 0 } // 以下都不报错
obj.aaa
obj.aaa = 12
obj()
const n: number = obj

# (3) 类型推论

两种场景：声明变量并初始化；决定函数返回值。

# (4) 类型断言

使用 as 关键字来指定更加具体的类型。

let alink = document.getElementById('link')
// alink.href 报错

// 使用 as 关键字来指定更加具体的类型
let alink2 = document.getElementById('link') as HTMLAnchorElement
alink2.href

// 不常用，react中会发生冲突
// let alink2 = <HTMLAnchorElement>document.getElementById('link')

查看对应标签的类型：console.dir($n) -> 最后的原型上有

# (5) typeof

可在类型上下文中引用变量或属性的类型（类型查询）

console.log(typeof 'hello') //string

let p = {
  age: 23,
  name: 'yyds',
}
function func(info: typeof p) {}
func({ age: 23, name: 'dss' })

注意：不可以查询函数调用的返回值类型

# 三、Ts 的高级类型

# (1) class 类

class Person {
  // 实例属性
  age: number
  sex = '男'
  // 构造函数(不可添加返回值类型)
  constructor(age: number, sex: string) {
    this.age = age
    this.sex = sex
  }
  // 实例方法
  handle(n: number): void {
    this.age *= n
  }
}

  1. 继承类：extends

    class PersonSon extends Person {
  name: 'yyds'
}
const p = new Person(12, '女')
const p2 = new PersonSon(23, '男')

  2. 实现接口：implement(Ts 特有的)

    interface Single {
  name: String
  sing(): void
}
class Singler implements Single {
  // 需要实现接口里的所有属性和方法
  name: 'yyds'
  sing() {
    console.log(1)
  }
}

  3. 类成员可见性修饰符：

    • public(默认)：公有的

    • protected：受保护的（仅对声明所在类和子类中可见，实例对象不可见）

    • private：私有的（仅对声明所在类可见）

    • readonly：只读（防止在构造函数之外对属性进行赋值）

      如果没有指定类型，会和字面量类型一样

      在接口、{}的对象类型中也可使用

# (2) 类型兼容性

Ts 采用结构化类型系统，类型检查关注的是值所具有的形状。

  1. class 类兼容性

    class Point {
  x: number
  y: number
}
class Point2D {
  x: number
  y: number
}
class Point3D {
  x: number
  y: number
  z: number
}
let p: Point = new Point2D()
// 成员多的可以赋值给成员少的
let p2: Point = new Point3D()

  2. 接口兼容性

    接口和类之间的兼容性类似于 class，class 与 interface 之间也可兼容

    interface _Point {
  x: number
  y: number
}
interface _Point2D {
  x: number
  y: number
}
interface _Point3D {
  x: number
  y: number
  z: number
}
let p1: _Point
let p2: _Point2D = { x: 2, y: 5 }
let p3: _Point3D = { x: 32, y: 23, z: 23 }
p1 = p2
p1 = p3
let p: _Point2D = new Point3D()

  3. 函数兼容性

    type F1 = (a: number) => void
type F2 = (a: number, b: number) => void
let f1: F1 = (num) => console.log(1)
// 参数少的可以赋值给参数多的（Js中省略用不到的参数）
let f2: F2 = f1
// 返回值类型，参数多的赋值给参数少的

# (3) 交叉类型(&)

组合多个类型为一个类型（常用于对象类型）

interface Person {
  name: string
}
interface Contact {
  phone: string
}
let obj: Person & Contact = {
  name: 'yyds',
  phone: 'dsdds',
}

交叉类型和继承的区别：

  • 相同点：都可实现对象类型的组合

  • 不同点：同名属性之间，处理类型冲突方式不同

    interface A {
  fn: (a: number) => number
}
// interface B extends A {
//   fn: (a: string) => number	// 报错提示类型不兼容
// }
interface B {
  fn: (a: string) => number
}
type C = A & B
let c: C = {
  fn(a: number | string) {
    return 23
  },
}
// c.fn()

# (4) 泛型和 keyof

保证类型安全前提下，让函数与多种类型一起工作。

function func<Type>(value: Type): Type {
  return value
}
func<number>(12) // 类型推断Type为字面量类型，func(12)
func<string>('yyds') // func('yyds')

  1. 泛型约束：缩窄类型取值范围

    // 1.
function fn<Type>(value: Type[]): Type[] {
  console.log(value.length)
  return value
}
fn([12, 23])

// 2. 添加约束
interface Ilength {
  length: number
}
function fn2<Type extends Ilength>(value: Type): Type {
  console.log(value.length)
  return value
}
// 有length属性的即可
fn2('dsdd')
fn2([32, 'g'])
fn2({ length: 10 })

// 3. 类型变量之间约束
function fn3<Type, Key extends keyof Type>(obj: Type, key: Key) {
  return obj[key]
}
fn3({ name: 'yyds', age: 24 }, 'name')
fn3(123, 'toFixed')
fn3('dsdsd', 2) // 2 为索引

    keyof：获取对象类型所有的键。

  2. 泛型接口：

    interface IdFunc<Type> {
  id(value: Type): Type
  ids(): Type[]
}

let obj: IdFunc<number> = {
  id(value) {
    return value
  },
  ids() {
    return [23, 12, 3]
  },
}

  3. 泛型类：

    class Count<NumType> {
  default: NumType
  add: (x: NumType, y: NumType) => NumType
  // 此时可省略类型不写
  constructor(value: NumType) {
    this.default = value
  }
}
// let cnt = new Count<number>()
let cnt = new Count(11)
cnt.default = 12

  4. 泛型工具类型：

    • Partial<Type>：构造一个类型，将 Type 所有属性设置为可选

      interface Props {
  id: number
  name: string
}
type PartialProps = Partial<Props>

    • Readonly<Type>：构造一个类型，将 Type 所有属性设置为只读

      interface Props2 {
  id: number
  name: string
}
type ReadonlyProps = Readonly<Props2>
let read: ReadonlyProps = {
  id: 12,
  name: 'yyds',
}
// read.id = 14

    • Pick<Type, Keys>：从 Type 中选择一组属性来构造新类型。

      interface Props3 {
  id: number
  name: string
  children: string[]
}
type PickProps = Pick<Props3, 'id' | 'name'>

    • Record<Keys, Type>：构造一个类型，属性键为 Keys，属性类型为 Type。

      type RecordObj = Record<'a' | 'b' | 'c', string[]>
let recordobj: RecordObj = {
  a: ['a'],
  b: ['b'],
  c: ['c'],
}

# (5) 索引签名类型和 索引查询类型

  1. 索引签名类型：无法确定对象有哪些属性时。

    interface Prop {
  [key: string]: number
}
let prop: Prop = { a: 13, b: 11 }

// JS数组简单实现
interface MyArray<Type> {
  [index: number]: Type
}
let arr1: MyArray<number> = [12, 32, 43]
arr1[2]

    Js 对象的键为 string。

  2. 索引查询类型：

    type Props = { a: number; b: string; c: boolean }
type TypeA = Props['a']
type TypeB = Props['a' | 'b']
type TypeC = Props[keyof Props]

# (6) 映射类型

基于旧类型创建新类型（对象类型），减少重复、提升开发效率。

映射类型只能在类型别名中使用,不能再接口中使用！！！

// 1. 根据联合类型创建
type Propkeys = 'x' | 'y' | 'z'
// type Prop = {x: number, y: number, z: number}
type Prop = { [Key in Propkeys]: number }

// 映射类型只能在类型别名中使用,不能再接口中使用
// 错误演示
// interface Props{
//   [Key in PropKeys]: number
// }

// 2. 根据对象类型创建
type Props = { a: number; b: number; c: number }
type Prop2 = { [Keys in keyof Props]: number }

泛型工具类型Partial<Type>实现原理：

interface Props {
  id: number
  name: string
}
type PartialProps = Partial<Props>

type Partial<T> = {
  [P in keyof T]?: T[P]
}

# 四、Ts 类型声明文件

用来给已存在的 Js 库提供类型信息，标准化内置 API 都有对应的类型声明文件。

  1. .ts 文件：编写程序代码
  • 既包含类型信息又可执行代码
  • 可编译为.js 文件后执行
  1. .d.ts 文件：为 JS 提供类型
  • 只包含类型信息
  • 不生成.js 文件，仅用于提供类型

DefinitelyTyped (opens new window)：提供高质量 TypeScript 类型声明。 Ts 官方文档 (opens new window)

下载对应的声明类型文件：npm i @types/react

VScode 默认会加载同名的 .d.ts 文件。
