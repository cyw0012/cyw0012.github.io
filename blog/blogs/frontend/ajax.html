# Ajax

# 一、服务器的基本概念与初始 Ajax

# (1) 概念

  1. 客户端与服务器：

    • 服务器：存放和对外提供资源的电脑。

    • 客户端：获取和消费资源的电脑。

  2. URL 地址（统一资源定位符）：

    用于标识互联网上每个资源的唯一存放位置，浏览器只有通过 URL 地址，才能正确定位资源的存放位置，从而成功访问到对应的资源。

    三大组成部分：

    • 客户端与服务器之间的通讯协议

    • 存放资源的服务器名称

    • 资源在服务器上具体的存放位置

  3. 网页的打开过程：客户端与服务器之间的通信为：请求——处理——响应(网页的每个资源都是)

    • 基于浏览器的开发者工具分析通信过程：

    打开 Chrome->打开网页->打开开发者工具->切换到 Network 面板->选中 Doc 页签->刷新查看通信

    • 网页资源：文字内容、image 图片、Audio 音频、Video 视频、数据（灵魂）

    • 请求数据需要用到 XMLHttpRequest 对象（简称 xhr），是浏览器提供的 js 成员。（let xhrObj=new XMLHttpRequest()）

# (2) 初始 Ajax

了解 Ajax：在网页中利用 XMLHttpRequest 对象和服务器进行数据交互。

  • get 请求：用于获取服务器端资源

  • post 请求：用于向服务器提交数据

  1. $.get(url,[data],[callback])：获取数据

    // 发起不带参数的get请求
$.get('http://www.liulongbin.top:3006/api/getbooks', function (res) {
  console.log(res) //res为服务器返回的数据
})
// 发起带参数的get请求
$.get(
  'http://www.liulongbin.top:3006/api/getbooks',
  {
    id: 1,
  },
  function (res) {
    console.log(res)
  }
)

  2. $.post(url,[data],[callback])：提交数据

    let data = {
  bookname: '皮卡丘',
  author: 'yyds',
  publisher: '江西出版社',
}
$.post('http://www.liulongbin.top:3006/api/addbook', data, function (res) {
  console.log(res)
})

  3. $.ajax()：获取、提交数据

    $.ajax({
  type: 'POST', //发起get或者post请求
  url: 'http://www.liulongbin.top:3006/api/addbook',
  data: {
    bookname: '皮卡丘',
    author: 'yyds',
    publisher: '江西出版社',
  },
  success: function (res) {
    console.log(res)
  },
})

# 二、form 表单与模板引擎

# (1) form 表单的基本使用

  1. 表单主要负责数据的采集功能

  2. 组成部分：表单标签、表单域、表单按钮

  3. <form>标签属性（规定把采集到的数据发送到服务器）

    • action：如果未指定，则提交到当前页面，否则提交到指定的 URL 地址。

    • target：规定在何处打开 URL

    • method：规定以何种方式提交数据，get 或 post。get 适合提交少量、简单的数据；post 适提交大量、复杂、或包含文件的上传（安全）。

    • enctype：规定在发送数据前如何对数据进行编码

      值 描述
      application/x-www-form-urlencoded 在发送前编码所有字符（默认）
      multipart/form-data 不对字符编码。在使用包含文件上传控件的表单时，必须使用该值。
      text/plain 空格转换为”+”加号，但不对特殊字符编码。(很少用)

      WARNING

      注：涉及文件上传必须让其为 multipart/form-data，否则为默认值即可。

  4. 表单的同步提交：

    • 概念：通过点击 submit 按钮，触发表单提交操作，跳转到 action URL。

    • 缺点：整个页面跳转到 action URL，用户体验差；页面之前的状态和数据会丢失。

    • 解决：表单负责采集数据，Ajax 负责将数据提交到服务器。

  5. 通过 Ajax 提交表单数据

    • 表单可以监听表单提交事件(submit)

    • 快速获取表单数据：每个元素一定要加 name 属性，且唯一

    <form action="" method="post">
 <input type="text" name="name" value="freely"><br>
 <input type="text" name="age" value="20"><br>
 <input type="text" name="city" value="beijing"><br>
 <input type="text" name="job" value="fe"><br>
 <input type="file" name="file">
</form>

// 1. serialize()
$('form').serialize()
// 调用结果：
// name=freely&age=20&city=beijing&job=fe

// 2. serializeArray()
$('form').serializeArray()
// 调用结果：
// 得到数组对象
/*
 0: {name: "name", value: "freely"}
 1: {name: "age", value: "20"}
 2: {name: "city", value: "beijing"}
 3: {name: "job", value: "fe"}
*/

// 两种方法文件选择元素都不会被序列化！

# (2) art-template 模板引擎

模板引擎( 地址 (opens new window) )：可以指定的模板结构和数据，自动生成一个完整的 html 页面。

好处：减少字符串拼接；结构清晰；便于阅读与维护。

  1. 使用步骤：

    • 导入 art-template

    • 定义数据(data)

    • 定义模板

      <script type="text/html" id="起个id名"></script>

    • 调用 template 函数（编译）

      let htmlStr = template('模板id', data)

    • 渲染 html 结构

  2. 语法：

    - 标准语法：`{{ }}`在内部可进行变量输出，或循环数组

- 输出：`{{ }}`内可进行变量、对象属性、运算结果输出

- 原文输出：`{{@ }}` 如果要输出的 value 值中有 html 标签结构，则需原文输出，才可正常渲染

- 条件输出：`{{if...}}...{{else if...}}...{{/if}}`

    • 循环输出：

      {{each arr}} {{$index}} {{$value}} {{/each}}

    • 过滤器：需要处理的值-》过滤器函数-》返回新值

      {
  {
    value | filterName
  }
} // | 管道符

      定义一个格式化时间的过滤器：

      template.defaults.imports.dateFormat = function (date) {
  let y = date.getFullYear()
  let m = date.getMonth() + 1
  let d = date.getDate()
  //过滤器最后一定要return一个值
  return y + '-' + m + '-' + d
}

      使用：

      <div>注册时间： {{regTime | dateFormat}}</div>

# (3) 模板引擎的实现原理

  1. 正则表达式匹配：exec()用于检索字符串中的正则表达式匹配

  2. 替换：replace

    let data = {
  name: 'cyw',
  age: 36,
}
let str = '<div>我是{{name}}今年{{ age }}</div>'
let pattern = /{{(\s*([a-zA-Z]+)\s*)}}/ //正则表达式

let patternRes = null
while ((patternRes = pattern.exec(str))) {
  str = str.replace(patternRes[0], data[patternRes[2]])
}
console.log(str)

# 三、Ajax 加强

# (1) XMLHttpRequest 的基本使用

  1. xhr 发起 get 请求

    // 1.创建XHR对象
let xhr = new XMLHttpRequest()
// 2.调用 open 函数
// 不带参
// xhr.open("GET", "http://www.liulongbin.top:3006/api/getbooks")
// 带参
xhr.open('GET', 'http://www.liulongbin.top:3006/api/getbooks?id=1')
// 3.调用 send 函数
xhr.send()
// 4.监听 onreadystatechange 事件
xhr.onreadystatechange = function () {
  if (xhr.readyState === 4 && xhr.status === 200) {
    //status与响应数据中的status不是同一个
    // 获取服务器响应的数据
    console.log(xhr.responseText)
  }
}

    readyState 属性：表示当前 ajax 请求所处的状态

    值 状态 描述
    0 UNSENT XMLHttpRequest 对象已被创建，但尚未调用 open 方法。
    1 OPENED open()方法已经被调用。
    2 HEADERS_RECEIVED send()方法已经被调用，响应头也已经被接收。
    3 LOADING 数据接收中，此时 response 属性中已经包含部分数据。
    4 DONE Ajax 请求完成，这意味着数据传输已经彻底完成或失败。

    带参的 get 请求：本质是将参数以查询字符串的形式，追加到 URL 地址后。

    WARNING

    不允许有中文，需要将中文字符进行编码（转义）；一个汉字编码成英文字符为三组%

    编码：encodeURI("好好学习")；解码：decodeURI('%E7%9A%AE%E5%8D%A1%E4%B8%98')

    浏览器会自动对 URL 地址编码操作

  2. xhr 发起 post 请求

    // 1.创建XHR对象
let xhr = new XMLHttpRequest()
// 2.调用 open 函数
xhr.open('POST', 'http://www.liulongbin.top:3006/api/addbook')
// 3.设置 Content-Type 属性(固定写法)
xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded')
// 4.调用 send 函数
xhr.send('bookname=皮卡丘&author=yyds&publisher=小陈出版社')
// 5.监听 onreadystatechange 事件
xhr.onreadystatechange = function () {
  if (xhr.readyState === 4 && xhr.status === 200) {
    //status与响应数据中的status不是同一个
    // 获取服务器响应的数据
    console.log(xhr.responseText)
  }
}

# (2) 数据交换格式

服务端与客户端之间进行数据传输与交换的格式有 XML（用的少）、JSON...

  1. XML(EXtensible Markup Language)：可扩展标记语言

    HTML 网页内容的载体；XML 网页数据的载体

    缺点：格式臃肿，体积大，传输效率低；js 解析 xml 麻烦

  2. JSON(JavaScript Object Notation)：js 对象和数组的字符串表示法，本质是字符串

    优点：轻量级的数据交换格式，更小、更轻、更易于解析。

    JSON 的两种格式：

    • 对象结构：{key:value,...}

      key 为双引号包裹的字符串；value 为数字、字符串、数组、布尔值、对象、null。

    • 数组结构：[value,...]

      value 为数字、字符串、数组、布尔值、对象、null。

      WARNING

      注意事项：

      • 属性名、值必须被双引号包裹

      • 不允许使用单引号表示字符串

      • 最外层必须为对象或数组

      • 不能用 undefined、函数作为 JSON 的值

      JSON 的作用：计算机与网络直接存储和传输数据

      JSON 的本质：用字符串表示 js 对象或数组

    JSON 和 JS 对象转换：

    • 字符串转 JS 对象（反序列化）：JSON.parse(json)

    • JS 对象转字符串（序列化）：JSON.stringify(obj)

# (3) XMLHttpRequest Level2 的新特性

  1. 旧版 XMLHttpRequest 的缺点：

    只支持文本数据的传输，无法读取、上传文件；没有进度信息。

  2. XMLHttpRequest Level2 的新功能：

    可设置 HTTP 请求时限；可以设置 FormData 对象管理表单数据；可以上传文件；可以获取进度信息。

    • HTTP 请求时限：

      xhr.timeout = 3000

xhr.ontimeout = function(event){
    alert('请求超时！')
}

    • 设置 FormData 对象管理表单数据：

    • 上传文件：向 FormData 追加文件。

      fd.append('avatar', files[0])

    • 获取进度信息：

      // 监听文件上传进度
xhr.upload.onprogress = function (e) {
  if (e.lengthComputable) {
    let progress = Math.ceil((e.loaded / e.total) * 100)
    console.log(progress)
  }
}
// 监听文件上传成功
xhr.upload.onload = function () {}

# (4) axios

专注于网络数据请求的库

相比于 xhr 对象，更简单易用；相比于 jQuery，更轻量化。

// axios返回的值给我们套了个壳，服务器返回的数据在data中，可使用解构赋值 {data: res}。
axios({
  method: 'GET',
  url: 'http://www.liulongbin.top:3006/api/getbooks',
  // URL查询参数，GET请求携带参数
  params: {
    id: 1,
  },
  // 请求体参数，POST请求携带参数
  data: {},
}).then((res) => {
  console.log(res)
})

如果方法的返回值为 Promise 实例，则前面可以添加 await；await 只能用在被 async 修饰的方法中。

注意在请求时，可能要设置 Content-Type！！！

// 在全局设置
const request = axios.create({
  baseURL: 'http://127.0.0.1',
  headers: {
    'Content-Type': ' application/x-www-form-urlencoded',
  },
})
// 单独设置
request.post(
  '/api/reguser',
  {
    username,
    password,
  },
  {
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  }
)

# 四、跨域与 JSONP

# (1) 同源策略与跨域

  1. 同源策略：两个页面的协议、域名、端口相同，两个页面具有相同的源。

    非同源的网站：

    • 无法读取非同源网页的 Cookie、LocalStorage、IndexedDB

    • 无法接触非同源网页的 DOM

    • 无法向非同源地址发送 Ajax 请求

  2. 跨域：与同源相反，出现的根本原因是浏览器的同源策略不允许非同源的 URL 数据交换。

    实现跨域请求：

    • JSONP：临时解决方案，只支持 GET 请求

    • CORS：根本解决方案，W3C 标准，支持 GET 和 POST 请求，但不兼容某些低版本浏览器。

# (2) JSONP

  1. 实现原理：通过<script>标签的 src 属性，请求跨域的数据接口，并通过函数调用的形式，接收跨域接口响应回来的数据。

    <script src="http://www.liulongbin.top:3006/api/jsonp?callback=success&name=ls&age=30"></script>

  2. 注意：JSON 与 Ajax 之间没有任何关系

  3. jQuery 中的 Ajax 来实现 JSONP 是通过动态创建<script>标签，然后请求成功又移除。

# (3) 防抖

事件触发后，延迟 n 秒后再执行回调，如果 n 秒内事件又被触发则重新计时。

目的：通过防抖策略，只在输入完成后执行查询请求，减少请求次数，节约请求资源。

减少请求次数，同时还可以缓存搜索建议。

# (4) 节流

减少一段时间内事件触发的频率。

应用场景：鼠标连续的触发某点击事件，只在单位时间内触发一次；懒加载时要计算滚动条的位置，但不必每次滑动都触发，可以降低计算频率，而不必浪费 CPU 资源。

TIP

防抖与节流的区别：

  • 防抖：只有最后一次触发生效，前面 N 多次触发被忽略。

  • 节流：触发一次事件后立即执行，在一段时间内再触发事件不生效。

# 五、HTTP 加强

通信三要素：主体、内容、方式

通信协议：网页内容的传输协议又叫超文本传输协议（简称 HTTP 协议）。

HTTP 协议规定了客户端与服务端之间进行网页内容传输时，所必须遵守的传输协议。

# (1) HTTP 请求与响应

  1. HTTP 请求

    客户端发送到服务端的消息为HTTP 请求消息，也叫请求报文。

    HTTP 请求消息的组成部分：

    • 请求头部：键值对组成。描述客户端的基本信息，告知给服务器。
    头部字段 说明
    Host 要请求的服务器域名
    Connection 客户端与服务器的连接方式(close 或 keepalive)
    Content-Length 用来描述请求体的大小
    Accept 客户端可识别的响应内容类型列表
    User-Agent 产生请求的浏览器类型
    Content-Type 客户端告诉服务器实际发送的数据类型
    Accept-Encoding 客户端可接收的内容压缩编码形式
    Accept-Language 用户期望获得的自然语言的优先顺序

    • 空行：用于分隔

    • 请求体（只有 POST 请求才有）：存放的是通过 POST 方式提交到服务器的数据。

  2. HTTP 响应

    服务端发送到客户端的消息为HTTP 响应消息，也叫响应报文。

    HTTP 响应消息的组成部分：

    • 响应头部：描述服务器的基本信息，告知给客户端。

    • 响应体：存放的是服务器发送给客户端的消息。

# (3) HTTP 请求方法

表面要对服务器上的资源执行的操作。

常见的请求方法：

方法 描述
GET (查询) 发送请求来获得服务器上的资源，请求体中不会包含请求数据，请求数据放在协议头中。
POST (新增) 向服务器提交资源（例如提交表单或上传文件）。数据被包含在请求体中提交给服务器。
PUT (修改) 向服务器提交资源，并使用提交的新资源，替换掉服务器对应的旧资源。
DELETE (删除) 请求服务器刷除指定的资源。
HEAD HEAD 方法清求一个与 GET 请求的响应相同的响应，但没有响应体。
OPTIONS 获取 http 服务器支持的 http 请求方法，允许客户端查看服务器的性能，比如 ajax 跨域时的预检等。
CONNECT 建立一个到由目标资源标识的服务器的隧道。
TRACE 沿着到目标资源的路径执行一个消息环回测试，主要用于测试或诊断。
PATCH 是对 PUT 方法的补充，用来对已知资源进行局部更新。

# (4) HTTP 响应状态码

响应状态码查询 (opens new window)

标识响应的状态，由三个十进制组成，第一个定义了状态码的类型，后面两个对状态码进行细分。

分类 分类描述
1** 信息，服务器收到情求，需要请求者继续执行操作（实际开发中很少遇到 1**类型的状态码）
2** 成功，操作被成功接收并处理
3** 重定向，需要进步的操以完成请球
4** 客户端错误，请求包含语法错误或无法完成请求
5** 服务器错误，服务器在处理请求的过程中发生了错误

# 六、Git

版本控制软件是一个用来记录文件变化，负责管理文件版本的软件。

使用版本控制软件的好处：操作简单（只需识记几组简单的终端命令）、易于对比、易于回溯、不易丢失、协作方便。

版本控制系统分类：

  • 本地版本控制系统

    优点：使用软件记录文件的不同版本，提高了工作效率，降低了手动维护版本的出错率。

    缺点：单机运行，不支持多人协作开发；版本数据库故障后，所有历史更新记录会丢失。

  • 集中化的版本控制系统：基于服务器、客户端的运行模式。服务器保存文件的所有更新记录；客户端只保留最新的文件版本。

    优点：联网运行，支持多人协作开发。

    缺点：不支持离线提交版本更新；中心服务器崩溃，所有人无法正常工作；版本数据库故障后，所有历史更新记录会丢失。

  • 分布式版本控制系统：基于服务器、客户端的运行模式。服务器保存文件的所有更新版本；客户端是服务器的完整备份，便不是只保留文件的最新版本。

    优点：联网运行，支持多人协作开发；客户端断网后支持离线本地提交版本更新；服务器故障或损坏后，可使用任何一个客户端的备份进行恢复。

# (1) 起步

Git 是一个开源的分布式版本控制系统，是目前最先进、最流行的版本控制系统，高性能，高可靠性。

  1. 两个特性：

    • 直接记录快照，而非差异比较

    SVN 是基于差异的版本控制，节省磁盘空间；耗时、效率低；Git 快照类似于备份，占用磁盘空间大；版本切换时非常快。

    • 近乎所有操作都是本地执行

    断网后可在本地对项目进行版本管理；联网后同步到云端服务器。

  2. Git 的三个区域：工作区、暂存区、Git 仓库

  3. Git 的三种状态：已修改(modified)、已暂存(staged)、已提交(committed)

  4. Git 工作流程：

# (2) Git 基础

  1. 安装 Git：Git 官网 (opens new window)

  2. 配置用户基本信息

    右键点击 Git Bash Here 进入操作面板，或者直接打开终端。

    $ git config --global user.name "yyds"

$ git config --global user.email "100861234@qq.com"

    使用了--global 选项，那么该命令只需要运行一次，即可永久生效。

  3. 全局配置文件：C:/Users/用户名文件夹/.gitconfig

  4. 检查配置信息：

    # 查看所有的全局配置
git config --list --global

# 查看指定的全局配置项
git config user.name
git config user.email

  5. 获取帮助信息：

    git help config

git config -h 获取极简信息

# (3) Git 基操

  1. 初始化：本地目录转为 Git 仓库

    创建了 .git 隐藏目录

    git init

  2. 检查文件状态

    git status
# 获取极简信息(git status --short)
git status -s

    ??：处于未跟踪状态。

    绿 A 标记：处于暂存状态。

    红色 M 标记：处于已修改状态

  3. 跟踪新文件：未跟踪的文件->暂存区

    # 一次性添加一个文件
git add 文件名
# 一次性添加多个文件
git add .

    注意：如果有子文件夹还是没有跟踪上，则有可能是该文件夹已经有 git 仓库了，需要把该文件夹目录下隐藏的.git 文件夹删除。

  4. 提交更新至仓库

    git commit -m "对提交内容的描述"

  5. 对已提交的文件进行修改

    暂存修改：git add 文件名——>提交至仓库：git commit -m "XXX"

  6. 撤销对文件的修改：还原为 Git 仓库所保留的版本

    git checkout -- 文件名

  7. 取消暂存文件

    # 移除单个文件
git reset HEAD 要移除的文件名称

# 移除多个文件
git reset HEAD .

  8. 跳过使用暂存区：工作区->Git 仓库

    git commit -a -m "描述信息"

  9. 移除文件

    # 从Git仓库和工作区同时删除文件
git rm -f 文件名

从Git仓库中移除，保留工作区文件
git rm --cached 文件名

  10. 忽略文件：创建一个名为.gitignore 的配置文件

    格式规范：以 # 开头为注释；以 / 开头为目录；以 ! 开头为取反；glob 模式。

    符号 说明
    * 匹配零个或多个任意字符
    [abc] 匹配任何在方括号中的字符
    ? 只匹配一个任意字符
    - 匹配两个字符范围内的字符
    ** 匹配任意中间目录
    展开 
     # 忽略所有的 .a文件
 *.a

 # 但跟踪所有的 1ib.a ,即便你在前面忽略了 .a文件
 !lib.a

 # 只忽略当前目录下的 TODO 文件，而不忽略 subdir/TODO
 /TODO

 # 忽略任何目录下名为 build 的文件夹
 build/

 # 忽略 doc/notes.txt ，但不忽略 doc/server/arch.txt
 doc/*.txt

 # 忽略 doc/ 目录及其所有子目录下的 .pdf文件
 doc/**/*.pdf

  11. 查看提交历史：(按 q 退出)

    # 按时间先后顺序列出所有的提交历史，最近的提交排在上面
git log

# 只展示最新的n条提交历史
git log -n

# 在一行上展示最新的n条提交历史
git log -n --pretty=oneline

# 自定义输出格式(提交的简写哈希值、作者名字、作者修订日期、说明)
git log -2 --pretty=format:"%h | %an | %ar | %s"

  12. 回退到指定的版本

    # 在一行上展示最新的n条提交历史
git log --pretty=oneline

# 根据指定的提交ID回退到指定版本
git reset --hard <COmmitID>

# 在旧版本中，查看全部命令操作的历史
git reflog --pretty=oneline

# 再次根据最新的提交ID，跳转到最新版本
git reset --hard <CommitID>

# (4) GitHub

引入：开源即提供程序和源代码；闭源即提供程序，不提供源代码。

五种开源协议：

  • BSD (Berkeley Software Distribution)

  • Apache Licence 2.0

  • GPL (GNU General Public License)

    • 具有传染性的一种开源协议，不允许修改后和衍生的代码做为闭源的商业软件发布和销售

    • 使用 GPL 的最著名的软件项目是：Linux

  • LGPL (GNU Lesser General Public License)

  • MIT (Massachusetts Institute of Technology,MIT)

    • 是目前限制最少的协议，唯一的条件：在修改后的代码或者发行包中，必须包含原作者的许可信息

    • 使用 MIT 的软件项目有：jquery、Node,js

开源项目提供平台：GitHub、Gitlab、Gitee。（只能托管以 Git 管理的托管平台）

两种访问方式：

  • HTTPS：零配置，每次访问仓库需输入账号密码

  • SSH：需额外配置，每次访问仓库无需重复输入账号密码

  1. 基于 HTTPS 将本地仓库上传到 GitHub

    # 第一次
# 本地仓库与远程仓库进行关联
git remote add origin https://github.com/yyds1234/module.git

# 上传
git push -u origin main

# 以后
git push

  2. 基于 SSH 将本地仓库上传到 GitHub

    作用是本地仓库与 GitHub 之间免登录的加密传输；由两部分组成，id_rsa(私钥文件，存放到客户端)、id_rsa.pub(公钥文件，需配置到 GitHub)

    (1) 生成：执行下面命令，再敲三次回车，到 C:\Users\用户名文件夹.ssh 中查看

    ssh-keygen -t rsa -b 4096 -C "100861234@qq.com"

    (2) 配置：

    • 用记事本打开 id_rsa.pub 文件，复制内容

    • 在浏览器中登录 Github,点击头像->Settings->SSH and GPG Keys->New SSH key

    • 将 id rsa.pub 文件中的内容，粘贴到 Key 对应的文本框中

    • 在 Title 文本框中任意填写一个名称，来标识这个 Key 从何而来

    (3) 检测配置是否成功：

    ssh -T git@github.com
# 然后输入 yes
# 出现这个表示成功：Hi yyds1234! You've successfully authenticated，but GitHub does notprovide shell access.

    (4) 上传

    # 第一次
git remote add origin git@github.com:cyw0012/moudle2.git

git push -u origin main

# 以后
git push -u origin main

    (5) 将远程仓库克隆到本地

    # -b master克隆指定分支、或者更改默认分支
git clone -b master git@github.com:cyw0012/moudle2.git

    (6) 切换仓库地址

    # 第一：查看远程库的信息
git remote -v

# 第二: 删除现有的远程仓库
git remote rm origin

# 第三: 建立新的远程仓库地址
git remote add origin + 远程仓库地址

# (5) Git 分支

main 主分支的作用：用来保存和记录整个项目已完成的功能代码。

功能分支：专门开发新功能的分支。

  1. 看分支列表：*表示当前分支

    git branch

  2. 创建分支：

    git branch 分支名称

  3. 切换分支：

    git checkout 分支名称

  4. 分支的快速创建和切换：

    git checkout -b 分支名称

  5. 合并分支：首先要切换到主分支，然后执行以下命令

    git merge 分支名称

  6. 删除分支：注意此时不能在删除分支上(-D 强制删除)

    git branch -d 分支名称

  7. 遇到冲突时合并分支：在不同的两个分支中，对同一个文件进行了修改，需要手动解决冲突，然后通知 git

    # 先手动解决冲突，在 VScode 中有提示
git add .
git commit -m "解决了分支合并问题"

远程分支操作：

  1. 将本地分支推送到远程仓库(如果远程仓库中有分支则不用加-u)：

    git push -u 远程仓库名称(默认为origin) 本地分支名称[:远程分支名称]

  2. 查看远程仓库的所有分支：

    git remote show origin

  3. 跟踪分支：远程分支下载到本地

    git checkout 远程分支名称（本地分支和远程分支名称相同）

git checkout -b 本地分支名称 origin/远程分支名称

  4. 拉取远程分支的最新代码

    git pull

  5. 将远程分支上的文件拉回来

    git pull origin main --allow-unrelated-histories

  6. 删除远程分支

    git push origin --delete 远程分支名称

  7. 删除远程分支的文件、文件夹

    将远程分支拉取回来 -> 移除指定文件或文件夹 -> 再重新将本地分支推送到远程仓库（会替换掉远程分支的内容，从而达到删除效果）
